Modularity

The boundaries between the upper layers (session, presentation and application) get particularly fuzzy. Some protocols are clearly designed to fit at one of these layers, while others may overlap all three.

It is for this reason that I do not categorize higher-level protocols by layer in this Guide. (The OSI Reference Model was in fact designed to take into account the fact that differentiating between these layers might not make sense.)

Key OSI Concepts

OSI Reference Model Layers
7 conceptual layers - ranking 1-7. Layer number represents position of layer in the model as a whole, based on closeness to hardware.
Lowest is physical layer, where low-level signaling and hardware is implemented.

Lower Layers (L1-4): Lower layers - physical, data link, network and transport.

Upper Layers (L5-7): Higher layers - session, presentation, application. Concerned with interacting with user.

N-notation

Each service uses services of layer N-1, and provides services to N+1.
If you're at Network layer (3) then N+2 is Session (layer 5).

Interfaces:
Vertical (Adjacent Layer) Communication - Communicating up/down a layer.

Generally use one of two transport layer (L4) protocols: TCP (transmission control protocol) and UDP (user datagram protocol).

Horizontal Communication:

Aside from Physical layer 1, horizontal communication also requires vertical communication.

To reach Layer 5 from one place to the other, must go from 5->1 then 1->5 after on the other machine.

Data Encapsulation:
Lower layers provide services to layers immediately above them. So service in each layer provides function to handle/manage data from the layer above.
Layer N PDU is passed to layer N-1, becoming the data that the layer N-1 protocol is supposed to be service.
Service Data Unit (SDU) places layer N SDU into its own PDU format, preceding the SDU with its own headers and appending footers.
This process is data encapsulation: entire contents of higher layer message are encapsulated as the data payuload of the message at the lower layer.

Data Encapsulation in TCP/IP

Transmission Control Protocol (TCP) operates at layer 4 of the OSI model. It transmits messages called segments that contain data encapsulated from higher-layer protocols.

The layer below TCP is the Internet Protocol (IP) at layer 3. It receives data from TCP and encapsulates it for transmission.
TCP segments are created as layer 4 PDUs. When passed to IP, they are treated as layer 3 SDUs.

These are in turn passed down to a layer 2 protocol, say Ethernet, which treats IP datagrams as layer 2 SDUs, and packages them into layer 2 PDUs (Ethernet frames) which are sent on layer 1. (Actually, in some technologies further encapsulation even occurs at layer one prior to transmission.)

On the receiving device, the process of encapsulation is reversed. The Ethernet software inspects the layer 2 PDU (Ethernet frame) and removes from it the layer 2 SDU (IP datagram) which it passes up to IP as a layer 3 PDU. The IP layer removes the layer 3 SDU (TCP segment) and passes it to TCP as a layer 4 PDU. TCP in turn continues the process, going back up the protocol layer stack. Physical Layer / Layer 1

Layer functions:

  • Definition of hardware specs.
  • Encoding and signaling
  • Data transmission and reception
  • Topology and physical network design

Raw bits.

Data Link Layer / Layer 2

Data link layer - where many wired and wireless Local Area Networking (LAN), technologies primarily function.
The set of devices connected at the data link layer is what is commonly considered a simple “network”, as opposed to an internetwork.

Turns raw bits into frames


Above shows headers of higher layers being added (MAC header, IP header, TCP header, HTTP header, Data)

The data link layer is often conceptually divided into two sublayers: logical link control (LLC) and media access control (MAC).

Tasks:

  • Logical Link Control - functions required for establishment and control of logical links between local devices on a network.
  • Media Access Control (MAC) - Refers to procedures used by devices to control access to network medium
  • Data Framing: final encapsulation of higher level messages into frames sent over network at physical layer
  • Addressing: Data link layer responsible for final encapsulation of higher level messages into frames sent over network at physical layer
  • Error Detection and Handling: data link layer handles errors that occur at lower levels of network stack.

Example: IEEE 802.3 specifies how ethernet works at data link layer (and also physical layers).
Very close relations between this layer and physical layer 1.

Network Layer / Layer 3

Network Layer:
Data Link defines boundaries of what is considered a network .
Network layer defines how internetworks function.

Routes data frames across different networks (IP)

  • Logical Addressing: Communication over network with it logical address sometimes called 'layer three' address. For example - IP is network layer protocol.
  • Routing: moving data across series of interconnected networks is probably defining function of network layer.
  • Datagram Encapsulation: layer normally encapsulates messages received from higher layers by placing them into datagrams (also called packets) - with network layer header.
  • Fragmentation and Reassembly: network layer must send messages down to data link layer for transmission, but if too large network layer must split it up. e.g. see Internet Protocol
  • Error handling and Diagnostics: special protocols are used at network layer to allow devices that are logically connected.

Most of the time Layer 3 is connectionless, then layer 4 connection-oriented. e.g TCP/IP, the Internet Protocol (IP) is connectionless, while the layer four Transmission Control Protocol (TCP) is connection-oriented.

The most common network layer protocol is of course the Internet Protocol (IP), which is why I have already mentioned it a couple of times. IP is the backbone of the Internet, and the foundation of the entire TCP/IP protocol suite

Transport Layer / Layer 4

Layers 1-3 are packaging, addressing, routing and delivery of data.
Data link Layer 2 with local networks, Layer 3 network layer handling routing between networks.
Transport is a liason between abstract world of applications at higher layers, and concrete functions of layers 1-3.

TCP / UDP. End to end communication between two nodes.
TCP divides the data to segments individually, with sequence numbers, and reassemble downstream.

Layer 4 also provide connection services for protocol and apps above it. Can be connection-oriented or connectionless services.

Session Layer / Layer 5

Practical matters related to addressing, package, deliver all left behind.
Allows devices to establish and manage sessions - the persistent logical linking of two software application processes.

Primary job of sesison layer protocol is to provide means to set up manage and end sessions. Session layer tools provided to higher layer protocols through command sets called APIs.

Presentation Layer (L6)

Limited and specific function than other layers.
To do with presentation of data. More specifically, the presentation layer is charged with taking care of any issues that might arise where data sent from one system needs to be viewed in a different way by the other system.

Examples:
Here are some of the specific types of data handling issues that the presentation layer handles:

Translation: Networks can connect very different types of computers together: PCs, Macintoshes, UNIX systems, AS/400 servers and mainframes can all exist on the same network. These systems have many distinct characteristics and represent data in different ways; they may use different character sets for example. The presentation layer handles the job of hiding these differences between machines.

Compression: Compression (and decompression) may be done at the presentation layer to improve the throughput of data. (There are some who believe this is not, strictly speaking, a function of the presentation layer.)

Encryption: Some types of encryption (and decryption) are performed at the presentation layer. This ensures the security of the data as it travels down the protocol stack. For example, one of the most popular encryption schemes that is usually associated with the presentation layer is the Secure Sockets Layer (SSL) protocol. Not all encryption is done at layer 6, however; some encryption is often done at lower layers in the protocol stack, in technologies such as IPSec.

Application Layer (L7)

Implements the functions performed by users to accomplish various tasks over the network.
