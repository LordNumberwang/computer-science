3.2 Flip Flops

Need something to remember one bit of information from time t-1, so it can be remembered at time t.

At 'end of time' t-1, such an ingredient can be at 1 of two states: 'remembering 0' or 'remembering 1'.
This ingredient remembers by 'flipping' between these possible states.
Gates that flip between two states are "Flip-Flops"

The Delta at the bottom: means it is a sequential chip that depends on time.
out[t] = in[t-1]

Implementation of D Flip Flop -

  • A primitive. Built from actual Nand gates. (loop achieving an 'unclocked flip-flop')
  • Isolation across time steps using a master-slave setup.

Can remember one bit ... now can build everything else.
General paradigm: Combo of remember info via D Flip Flop, and manipulate them with combinatorial logic.

  • Array of DFFs (memory of your system) feeds into...
  • combinatorial logic with the new input...
  • which changes state of DFF in next time unit.

Remembering Forever:
Goal: 1-bit register until requested to load a new value.

How build this from D Flip Flop?

Time unit 2: Should be 1 output
Output only can change if load = 1.

Working Implementation:

MUX / Multiplexor used to determine whicih value to send through. If load, then 'in' otherwise... equal to previous 'out'.

RAM Unit

RAM Abstraction: a sequence of n addressable registers. Addresses 0 to n-1.
At any given point of time, only one register in RAM is selected.

k (width of address input) => k = log2n
Use an input 'address' k.

w (word width) => w=16 for Hack computer.

RAM is sequential chip with clocked behavior.

3.4 Counters

For a program, tell when to go to what line of code.

Computer must keep track of which instruction should be fetched and executed next.
Control mechanism realized by a Program Counter.
PC contains the address of instruction that will be fetched and executed next.

Counter: Realizes this abstraction.
3 possible control settings:

  1. Reset: fetch the first instruction (PC=0)
  2. Next: fetch the next instruction (PC++)
  3. Goto: fetch instruction n

load, inc (increment), reset.

if (reset[t]==1) out[t+1] = 0 //reset, counter = 0
else if (load[t]==1) out[t+1] = in[t] //set counter = val.
else if (inc[t] == 1) out[t+1] = out[t]+1 //incr counter++;
else out[t+1] = out[t] //counter doesn't change
